# 📜 Protocolo: Universal Modularidade Genius (Ultra-LEGO Standard)

Este protocolo define a arquitetura obrigatória para qualquer unidade dentro do Ecossistema Genius (**Sistema, Módulo, Submódulo ou Fractal**). O princípio fundamental é a **Soberania Fractal**: cada unidade deve ser organizada como se fosse um sistema independente, capaz de ser plugado em qualquer plataforma com autonomia total.

---

## 🏛️ 1. Princípio da Unidade Independente
Toda pasta, independentemente do seu nível hierárquico, deve ser tratada como um "Sistema em Potencial". Isso significa que um Fractal deve ter a mesma maturidade organizacional de um Sistema Holding.

## 📂 2. Estrutura de Pastas (Quatrilogia de Soberania)

Toda unidade (**S/M/SM/F**) deve possuir, ou estar preparada para possuir, as seguintes subpastas:

1.  **`1_DNA_Unidade/`**: Identidade, Blueprints e Regras.
2.  **`2_Base_Dados/`**: Persistência, Esquemas e Logs.
3.  **`3_Interface/`**: Visualização e Interação (Frontend).
4.  **`4_Agentes/`**: DNAs dos agentes especialistas daquela unidade.

---

## ⚖️ 3. Regra de Unicidade e Independência
*   **Autonomia**: O sistema deve ser capaz de funcionar (ou ser migrado) apenas com o conteúdo desta pasta e as tabelas core do Supabase.
*   **Links Relativos**: Sempre que possível, utilize links que apontem para dentro da própria estrutura do sistema.
*   **SSOT**: A pasta `1_DNA_Sistema` é a única Fonte da Verdade para o funcionamento daquela unidade.

---

## 🤖 4. Responsabilidade dos Criadores
*   **Agente #06 (Criador de Módulos)**: Deve usar esta estrutura ao gerar o "esqueleto" de novos sistemas.
*   **Agente #05 (Criador de Agentes)**: Deve depositar o DNA dos agentes criados na pasta `4_Agentes` do sistema correspondente.
*   **Agente #38 (ClickUp)**: Deve registrar as tarefas de criação seguindo estes marcos (milestones).
